# Autodraw web (Astro + React)

Marketing site, `.adraw` spec, share viewer (`/v`), and the in-browser editor (`/app`) built with [Astro](https://astro.build/), [React islands](https://docs.astro.build/en/guides/integrations-guide/react/), and [Tailwind CSS v4](https://tailwindcss.com/).

## Develop

From the monorepo root:

```bash
pnpm --filter web dev
```

Or from `packages/web`:

```bash
pnpm dev
```

Open the URL Astro prints (default `http://localhost:4321`).

## Build

```bash
pnpm --filter web build
```

Output is static HTML + assets in `dist/`.

## Deploy on Cloudflare Workers (Workers Assets)

This app is configured for [Cloudflare Workers static assets](https://developers.cloudflare.com/workers/framework-guides/web-apps/astro/) (`wrangler.jsonc`, `preview` / `deploy` scripts).

From the monorepo root:

```bash
pnpm deploy:web
```

Or from `packages/web`:

```bash
pnpm preview   # astro build + wrangler dev
pnpm deploy    # astro build + deploy (requires `wrangler login`)
```

1. Copy `.dev.vars.example` to `.dev.vars` for local Workers preview when using env vars.
2. Edit `wrangler.jsonc` `name` to match your Worker name on Cloudflare.
3. Optional: `pnpm cf-typegen` generates `cloudflare-env.d.ts` for Wrangler env typings.

## Routes

| Path       | Notes                                      |
| ---------- | ------------------------------------------ |
| `/`        | Marketing landing + diagram hero preview |
| `/app`     | Full editor (`EditorShellLicensed`)      |
| `/spec`    | `.adraw` v1 format documentation           |
| `/v?d=…`  | Read-only viewer (payload decoded client-side) |
| `/imprint` | Legal placeholder                          |
| `/privacy` | Privacy placeholder                        |

Sitemap is generated by `@astrojs/sitemap` at build time (`/sitemap-index.xml` or `/sitemap-0.xml` depending on Astro version). Update `public/robots.txt` if the sitemap URL differs.
